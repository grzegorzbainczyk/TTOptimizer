About TTOrganizer

A timetable optimization prototype built with ASP.NET Core, C++, Entity Framework Core and JavaScript.

Project overview

TTOrganizer is a prototype web application for generating and presenting optimized school timetables. The project combines a web interface, an ASP.NET Core backend, a database layer and an external C++ optimization engine.

The main goal of this project is to build a practical end-to-end application while improving skills in modern backend development, C++ integration, frontend development, data exchange formats and deployment preparation.

What the application does

The application allows the user to manage school-related resources such as teachers, classes, subjects, rooms and lesson requirements.

Based on these data, the ASP.NET Core backend prepares an optimization problem and sends it to the external C++ engine. The engine generates a timetable result and returns it as JSON. The web application then maps technical identifiers to human-readable names and displays the timetable in a filterable table.

Architecture

Browser HTML / CSS / JavaScript
→
ASP.NET Core API Controllers / Services / DTOs
→
C++ Engine Optimization logic
→

Frontend Table Filters / CSV export / localStorage

JSON is used as the contract between the ASP.NET Core backend, the C++ engine and the browser. This separation keeps the optimization logic independent from the web interface.

C++

External optimization engine responsible for timetable generation logic.

Entity Framework Core

Database access, domain models, migrations and demo data handling.

JavaScript

Frontend behavior, API calls, table rendering, filtering, CSV export and localStorage.

HTML / CSS

Application layout, dashboard structure, tables, forms and responsive styling.

JSON

Data exchange format between the web backend, the C++ engine and the browser.

Implemented features

  • Demo login flow.
  • Management pages for teachers, classes, rooms, subjects and lesson requirements.
  • ASP.NET Core backend connected to a C++ optimization engine.
  • JSON input and output pipeline between backend and C++ engine.
  • Human-readable timetable result with teacher, class, subject and room names.
  • Filtering timetable results by class, teacher and room.
  • CSV export for the generated timetable.
  • Saving the last optimization result in browser localStorage.

What I learned

While working on this project, I practiced building an end-to-end application across multiple layers: frontend, backend, database and an external C++ engine.

I learned how to use JSON as a contract between separate technologies, how to call an external process from ASP.NET Core, how to map technical output to user-friendly data and how to build interactive frontend features with vanilla JavaScript.

A particularly important part of the project was separating responsibilities: the C++ engine focuses on optimization, the ASP.NET Core backend prepares and maps data, and the browser displays the result in a user-friendly way.

Current status

The application is currently a working prototype. It supports demo data, timetable generation, filtering, CSV export and local result persistence.
